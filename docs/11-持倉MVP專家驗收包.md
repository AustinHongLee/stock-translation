> 本文件是給懂股票的人驗收用，不要求產品主懂股票。

## 3. 公式

### 3.1 買進

```
買進總成本 = 股數 * 成交價 + 手續費
買進後股數 = 原股數 + 買進股數
買進後成本總額 = 原成本總額 + 買進總成本
平均成本 = 買進後成本總額 / 買進後股數
```

### 3.2 賣出

本版採移動平均成本法。

```
賣出收入 = 股數 * 成交價 - 手續費 - 證交稅
賣出對應成本 = 賣出股數 * 賣出前平均成本
已實現損益 = 賣出收入 - 賣出對應成本
賣出後股數 = 原股數 - 賣出股數
賣出後成本總額 = 原成本總額 - 賣出對應成本
```

限制：

- 不允許賣出超過目前持股。
- 賣出後股數為 0 時，成本總額歸 0。
- 不處理融資、融券、零股特殊費率。

### 3.3 未實現損益

```
市值 = 目前股數 * 最近收盤價
未實現損益 = 市值 - 成本總額
未實現報酬率 = 未實現損益 / 成本總額
```

限制：

- 最近收盤價不是即時價格。
- 若本地沒有日線資料，市值與未實現損益顯示為無法估算。
- 本版未納入股利、除權息與稅務。

## 4. 固定測試範例

### 範例 A：兩筆買進

輸入：

| 日期 | 買賣 | 股數 | 價格 | 手續費 |
|------|------|------|------|--------|
| 2026-01-02 | 買進 | 1000 | 900 | 100 |
| 2026-02-03 | 買進 | 1000 | 1000 | 120 |

期望：

- 股數：2000
- 成本總額：1,900,220
- 平均成本：950.11
- 若最近收盤 1058，市值：2,116,000
- 未實現損益：+215,780
- 未實現報酬率：約 +11.36%

### 範例 B：買進後部分賣出

輸入：

| 日期 | 買賣 | 股數 | 價格 | 手續費 | 證交稅 |
|------|------|------|------|--------|--------|
| 2026-01-02 | 買進 | 1000 | 900 | 100 | 0 |
| 2026-02-03 | 買進 | 1000 | 1000 | 120 | 0 |
| 2026-03-04 | 賣出 | 500 | 1100 | 80 | 1650 |

期望：

- 剩餘股數：1500
- 剩餘成本總額：1,425,165
- 平均成本：950.11
- 已實現損益：+73,215
- 若最近收盤 1058，未實現損益：+161,835

### 範例 C：賣超

輸入：

- 買進 100 股
- 賣出 101 股

期望：

- 系統拒絕儲存
- 錯誤原因：賣出股數超過目前持股

## 5. 驗收問題

請股票專家逐項檢查。

- [ ] 買進總成本包含手續費是否符合預期？
- [ ] 賣出收入扣除手續費與證交稅是否符合預期？
- [ ] 第一版使用移動平均成本法是否可接受？
- [ ] 未實現報酬率分母用成本總額是否合理？
- [ ] UI 是否需要更明顯標示「最近收盤價，不是即時價格」？
- [ ] 「帳面約賺 / 帳面約虧」是否比「賺 / 虧」更不易誤導？
- [ ] 未納入股利與除權息的限制是否足夠明顯？
- [ ] 哪個缺口最應該排入下一版？

---

## 6. 給小白產品主的驗收方式

產品主不需要判斷公式，只需要看：

- 是否知道這頁在看自己的持倉
- 是否知道帳面損益還不是落袋
- 是否知道價格不是即時價格
- 是否知道系統沒有叫你買或賣
- 是否知道可以把本文件交給懂股票的人檢查
