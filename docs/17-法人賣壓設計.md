# 17. 法人賣壓（籌碼面）設計 — 先設計、再串接

> 對齊《09》8.2 `chips/`（三大法人白話化）與本輪「地雷風險雷達」。本文是**設計＋待辦**，
> 演算法（純函數）已先放在 `app/chips/institutional.py`，資料來源待串。

## 0. 紅線
只用**公開資料的 proxy** 描述「法人在買還是在賣」這個事實；
**不臆測**「敏感型投控／高風險投控」之類的標籤、**不預測股價、不給買賣建議**。

## 1. 資料來源（待串接）
| 市場 | 來源 | 內容 |
|------|------|------|
| 上市 | TWSE 三大法人買賣超日報 | 每日 外資/投信/自營商 買賣超股數 |
| 上櫃 | TPEx 三大法人買賣明細 | 同上 |

串接方式延續既有 `app/sync` 轉接器模式：新增 adapter 抓每日三大法人資料 → 存本地表
`institutional_flows(stock_id, date, foreign_net, trust_net, dealer_net)` → 分析層用
`app/chips/institutional.py` 的純函數算 proxy。**外部資料只進 sync，分析層純函數可單測。**

## 2. 指標（公式，已實作為純函數）
正值＝淨賣超（賣壓），負值＝淨買超。
- **單一法人別賣壓** `selling_pressure = 淨賣超股數 / 當日成交量`
- **三大法人合計賣壓** `total_institutional_pressure = (外資+投信+自營商 淨賣超，只算賣方) / 當日成交量`
- **連續賣超天數** `consecutive_sell_days`（從最近一天往回數）
- **賣壓加速** `pressure_accelerating = 近 3 天平均賣超 > 近 20 天平均賣超`

## 3. 白話 proxy（中性、不臆測）
- 外資：外部/快錢資金，連續賣超 = 資金撤退訊號之一。
- 投信：本土法人、偏中長線，連續賣超 = 本土法人轉保守。
- 自營商：短線/避險為主，單看參考性較低。
- 三大法人同步賣超：盤面資金共識偏保守。

## 4. 接進「地雷風險雷達」
籌碼面是雷達的第 3 維度（風險矩陣已預留 `法人籌碼` 維度，目前 0 分）。
串好資料後：把「連續賣超 / 賣壓加速 / 三大同步賣」轉成 `法人籌碼` 維度分數，
併入 `risk_summary`，小白層「籌碼面」那行就能從『待接入』變成實際描述。

## 5. 施工順序
1. sync adapter 抓三大法人日資料 + 本地表。
2. `chips/institutional.py` 已就緒（純函數＋測試）。
3. 把籌碼分數接進 `risk_matrix` 的 `法人籌碼` 維度與 `risk_summary`。
4. UI：小白層「籌碼面」顯示連續賣超天數/賣壓；老手層給每日明細。

## v2 已實作（2026-06）：法人籌碼維度點亮

資料串接已完成，不再只是 proxy：

- **資料來源**：TWSE 每日「三大法人買賣超日報」(T86, `rwd/zh/fund/T86`, `selectType=ALLBUT0999`)。
  欄位對應：外資=第4欄、投信=第10欄、自營商=第11欄、三大法人合計=第18欄（正=買超、負=賣超）。
- **抓取**：`TwseClient.fetch_institutional_trades(stock_id, start, end, max_days=20)` 由近往遠逐日抓、跳過週末、連續無資料即停；同步時 best-effort 串入，失敗不影響整體同步。
- **儲存**：`institutional_trades` 表（`stock_id, date, foreign/trust/dealer/total_net`）。
- **分析（純函數）**：`build_institutional_summary()` 產生近 20 日趨勢、連續賣超天數、籌碼面等級（無／留意／注意／警戒）與白話摘要；所有對外字串通過禁字檢查。
- **呈現**：個股頁「法人籌碼」卡片（四大別買賣超＋連續賣超＋20日合計＋迷你走勢）；地雷雷達「籌碼面」那行改用真實資料。
- **紅線**：只描述買賣超事實，不預測股價、不給買賣建議；買賣超用中性藍／琥珀色，紅綠僅保留給漲跌（K 線）。

待辦：上櫃（TPEx）三大法人；融資融券、借券賣出等其他籌碼指標。
