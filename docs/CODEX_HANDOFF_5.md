## 0. 一句話

在**大型 K 線圖**裡加一顆按鈕「**讀圖導覽**」。按下去，App 就像一個技術派站在你旁邊，**一步步把腦內讀這張圖的過程念出來**：高亮某條線/某根 K/某個區域 → 跳出白話解說（這是什麼、為什麼看它、但要注意什麼）→ 最後畫出「**接下來會盯的關卡（條件式，非預測）**」。目標：**讓技術派的上手門檻降到幾乎為 0**。

---

## 1. 核心原則（先讀）

1. **教讀圖，不給訊號**：全程是「技術派傳統上會這樣看 + 但要注意」，**不是**「該買/該賣/會漲到」。
2. **重用既有資料**：導覽**不新增任何指標計算**。它把**已經算好**的東西（assessment 因子、levels 關卡、events 事件、indicators、structure 指紋、historical_frequency 情境、chips 法人、revenue/financial 基本面、news 風險）排成**有序分鏡（beats）**。
3. **後端產生分鏡腳本**：在 `app/analyze/chart_tour.py` 由既有 payload 組裝 beats，**旁白文字集中在後端**（可被紅線測試掃、可單元測試），前端只負責「播放」。
4. **資料缺就跳過**：某章節沒有資料（如沒下載法人）→ 自動略過該章，不報錯、不留空。
5. **預期線 = 條件式關卡**：唯一接近「未來」的地方，**硬規則**見 §5。

## 1.5 ⚠️ 真‧小白版旁白（最高優先；覆蓋下方 §2/§3/§9 的 narration 設計）

> 這段是這份文件**最重要**的修訂。原本的旁白（例：「KD 目前 K 58.05 / D 59.52；RSI 約 47.45；MACD DIF 約 0.24」）**完全是寫給有底子的技術派看的，小白看了等於零**。導覽的唯一目標是：**一個從沒看過股票的人，看完會說「啊我看懂了」**。

### 1.5.1 三層旁白（Beat.narration 改成這個結構）
每個 beat 的 `narration` 改為三層，**預設只顯示 Layer 0**：

```python
"narration": {
  "plain": str,     # Layer 0：生活白話。【硬規則】不准出現數字、不准出現未解釋的術語。一個 12 歲小孩要看得懂。
  "why":   str,     # Layer 1（點「為什麼」展開）：技術派為什麼看它，仍用白話。
  "detail":str      # Layer 2（點「看細節/數字」展開）：這裡才放 KD/RSI/MACD 數字與術語名。
}
```

- 前端**預設只播 Layer 0**；泡泡上有兩個小連結「為什麼?」「看數字」分別展開 Layer 1 / Layer 2。
- **小白全程只需要 Layer 0 就懂。** 數字與術語是給好奇的人「往下挖」用的，不是主角。

### 1.5.2 比喻庫（`chart_tour` 內建一張對照表，Layer 0 一律用左欄講法）
| 概念（術語） | Layer 0 只能這樣講（生活比喻） |
|---|---|
| 均線(MA/SMA/EMA) | 「平均價畫成的線」——把過去幾天的價格平均、連成一條，看趨勢不被單日大跳動騙。 |
| 短/中/長期均線 | 橘=短期人氣(近5天)、藍=中期成本(近20天)、紫=長期方向(近60天)。 |
| EMA vs SMA | 「都是平均線，EMA 對最近的價格反應快一點」（只在 Layer 2 提名字）。 |
| 黃金交叉 | 「短期的線**從下往上追過**中期的線」——像賽跑時後面的人超過前面。 |
| 死亡交叉 | 「短期的線被中期的線**蓋過去**」。 |
| 帶動/動態支撐 | 「價格一直**沿著**某條線上方走、回到附近又彈起來，那條線像在**撐著**它（像斜坡護欄）」。 |
| 乖離 | 「價格離平均線**拉太開**，像橡皮筋拉很長」。 |
| 支撐/壓力 | 地板(撐著)／天花板(壓著)。 |
| 成交量 | 「熱度——今天有多少人在買賣」。量縮=冷清、爆量=突然很多人。 |
| 動能(KD/RSI) | 「力氣」——會不會**太熱(追高風險高)**或**跌太深(超賣)**。 |
| 跳空 | 「開盤直接跳一格，跟昨天**沒連起來**」。 |
| 突破 | 「**衝破天花板並站穩**」。 |
> 比喻庫沒有的術語，Layer 0 **不准出現**。要用就先用一句話解釋。

### 1.5.3 開場要「從零」介紹圖本身（新增到 `intro` 章）
小白可能連 K 棒、線是什麼都不知道。開場前兩個 beat 先教：
- **beat `intro.candle`**：高亮一根 K 棒，Layer 0：「這張圖每一根代表**一天**：身體越長代表那天漲跌越多，紅色是漲、綠色是跌。」
- **beat `intro.lines`**：三條均線**依序發光**並標名，Layer 0：「圖上這幾條彩色的線叫『均線』，就是**過去幾天的平均價連成的線**，幫你看大方向。橘=短期、藍=中期、紫=長期。」

### 1.5.4 你點名要的：均線「帶動與交叉」橋段（在 `trend` 章，動畫演）
至少 3 個 beat，**鏡頭要移到事件點、圈起來**：
- **beat `trend.fast_slow`**：高亮橘線與藍線，Layer 0：「先看短的橘線跟中的藍線**誰在上面**。橘線在上＝最近比較熱、有人願意出更高價；橘線在下＝最近轉冷。」
- **beat `trend.cross`**（若視窗內有交叉，鏡頭 zoom 到交叉點＋圈起來）：Layer 0：「看這個**交叉點**——短期的橘線從下面**追過**中期的藍線。技術派叫它『黃金交叉』，白話就是『短期力量開始贏過中期』。⚠ 這只是描述**已經發生**的事，不代表之後一定會漲。」（沒交叉就講目前是誰在上、糾結在一起）
- **beat `trend.ride`**：高亮價格沿著某條均線走的那段，Layer 0：「你看價格這陣子都**沿著藍線上方**走、靠近又彈開——那條線像在**撐著**價格。」

### 1.5.5 把壞例子改成好例子（`momentum` 章示範）
- ❌ 原本：「KD 目前 K 58.05 / D 59.52；RSI 約 47.45；MACD DIF 約 0.24」
- ✅ Layer 0：「**動能就是現在有沒有力氣、會不會太熱。** 這檔現在力氣普通、還稍微偏冷一點——沒有過熱(追高比較危險)，也沒有跌到超賣。」
- ✅ Layer 1（為什麼）：「技術派看力氣，是想知道現在是不是漲過頭或跌過頭的極端區。」
- ✅ Layer 2（看數字）：「KD 的 K 約 58、D 約 60；RSI 約 47;MACD 的 DIF 約 0.24。」← 數字只活在這裡。

### 1.5.6 各章 Layer 0 黃金範本（Codex 照這個語氣寫，全部零數字零術語）
- `intro`：「我們像一個老手一樣，從頭把這張圖讀一遍，慢慢來。」
- `position`：「現在價格大概在過去一年的**中間**，不算貴也不算便宜（這只是位置，不是它值多少）。」
- `levels`：「上面這條是**天花板**：以前漲到這附近常上不去；下面這條是**地板**：跌到這附近常有人接。」
- `volume`：「**熱度**最近變低，關注的人比較少、有點冷清。」
- `events`：「← 這一天**突然跳了一格**開盤」「← 這天**特別多人交易**」。
- `chips`：「最近**外資**這群人是偏買還偏賣」（法人=大戶，用『一群大戶』講）。
- `fundamental`：「順便瞄一眼公司本身：最近營收比去年同期**多/少**、賺錢趨勢往上/往下。」
- `structure`：「這檔最近的『性格』：走勢比較**有延續性/比較亂**、波動比較**集中/分散**。」
- `scenario`：「最後看『**以前長得像現在的時候，後來那幾天跑去哪**』——這個扇形就是那些歷史結果的範圍。是歷史統計、**不是預測**，越往右越寬＝越不確定。」
- `watch`：「接下來只要記兩條線：**站上天花板**就看更上面、**跌破地板**就看更下面；沒突破前就是在中間晃。」
- `outro`：「這就是老手讀圖的順序。記得：這是**教你看懂**，不是叫你買賣。」

### 1.5.7 新增驗收（小白可懂的硬底線）
在 `tests/test_chart_tour.py` 加：
- **Layer 0 lint**：每個 beat 的 `narration.plain` **不得含阿拉伯數字 `[0-9]`**，且**不得含術語縮寫**（`KD|RSI|MACD|DIF|EMA|SMA|ATR|HV|OBV|β|MA\d`）——除非該詞在比喻庫且當場用白話解釋。違反即測試失敗。
- 數字/術語只能出現在 `narration.detail`（Layer 2）。
- 仍套原紅線字表（會漲/該買/目標價/前兆…）。

> 一句話給 Codex：**Layer 0 寫得像在跟你阿嬤解釋，數字全部滾去 Layer 2。**

## 2. Beat（分鏡）資料結構

`build_chart_tour(payload) -> {"available": bool, "beats": [Beat], "disclaimer": str}`

```python
Beat = {
  "id": str,                 # e.g. "trend.ma_alignment"
  "chapter": str,            # 見 §3 章節 key
  "title": str,              # 短標題（章節抬頭用）
  "targets": [Target],       # 要在圖上高亮的對象（可多個）
  "narration": {
     "headline": str,        # 一句白話結論
     "why": str,             # 技術派為什麼看它（傳統解讀）
     "caution": str          # 但要注意什麼（保持中性、非預測）
  },
  "source": str,             # 來源模組/欄位（可追溯，e.g. "assessment.ma")
  "confidence": "high"|"medium"|"low"   # 由資料充足度帶（資料少就標、降視覺權重）
}

Target =
  {"type":"ma", "key":"ma20"} |                         # 高亮一條疊圖均線（描出+發光）
  {"type":"level", "price":181.0, "label":"波段壓力", "role":"resistance",
   "origin_date":"2026-01-08"} |                         # 水平線 + 在形成它的那根 K 點圓點
  {"type":"region", "from_date":"...","to_date":"..."} | # 框/陰影一段日期（位階區、計算視窗）
  {"type":"candles", "dates":["2025-12-17","..."]} |     # 在指定 K 棒上掛箭頭/標記
  {"type":"subplot", "key":"kd"|"rsi"|"macd"|"volume"} | # 閃一下某副圖
  {"type":"scenario_cone"} |                             # 歷史情境扇形（已實作）
  {"type":"watch_level", "price":181.0, "label":"上方下一關卡",
   "condition":"break_above"|"break_below", "next_price":190.0}  # 預期線（條件式，§5）
```

## 3. 章節目錄（料要足——把現有資料全用上）

> 固定章節順序如下；**每章只在資料可用時出現**；每章 1–3 個 beat；整段總 beat 數上限 **~14**（避免太長）。每個 beat 的旁白**來源**標在右欄，**直接重用、不要新算**。

| # | 章節 key | 演什麼（高亮 + 旁白要點） | 旁白/資料來源 |
|---|---|---|---|
| 1 | `intro` | 開場：代號/名稱/產業、最近收盤+日期、一句話體質總評。「我們像技術派一樣讀這張圖。」 | `profile` / `summary` / `assessment.summary` |
| 2 | `trend` | 依序描出 MA5/20/60(/120)，講多頭/空頭/糾結、價在均線上下、均線斜率。 | `assessment.ma`、indicators `ma*`、`ma*_slope` |
| 3 | `position` | 框出近一年高低區，講價格位階(%)、距 52 週高/低、乖離(price vs MA20)。 | `assessment.bias`、`distance_to_52w_high/low`、`price_position` |
| 4 | `levels` | 閃出波壓/波撐(短/波段/長)，並在**形成關卡的那根 K** 點圓點；講「這是過去轉折/成交密集帶」。 | `levels.py`（修正版）、rolling `high_*/low_*` |
| 5 | `momentum` | 閃 KD/RSI（位置+金叉/死叉）、MACD、布林位置/收斂。 | `assessment.kd/rsi`、indicators `kd_*/rsi_*/macd*/bb_*` |
| 6 | `volume` | 閃量能柱：量比/爆量/量縮、量價關係(價漲量增…)、OBV 方向。 | `assessment.volume`、indicators `volume_ratio/obv/price_*_volume_*` |
| 7 | `events` | 在最近幾根 K 掛箭頭：跳空/突破/假突破/長黑長紅、除息/財報/營收事件。 | 本地事件、`gap_*`、`breakout_*`、events markers |
| 8 | `chips` | 三大法人近期方向與連續性（外資/投信/自營買賣超）。 | `chips`（institutional） |
| 9 | `fundamental` | 給技術派的背景一瞥：月營收 YoY、EPS/毛利趨勢、估值殖利率位階。**簡短**。 | `revenue`、`financial`、`valuation` |
| 10 | `structure` | 結構指紋一句話：延續性/複雜度/湍流/噪音色/波動聚集（接 HANDOFF_3 成果）。 | `structure`（structure_metrics） |
| 11 | `scenario` | 畫歷史情境扇形：「過去類似情況後的範圍（**非預測**）」。 | `historical_frequency` |
| 12 | `watch` | **接下來會盯什麼**：條件式關卡線（突破X看Z／跌破Y看W、動態均線支撐）＋消息面風險提醒。見 §5。 | `levels`、rolling、`news` 風險 |
| 13 | `outro` | 收尾：重申「這是讀圖識讀、不是買賣建議」＋資料日期/充足度提醒。 | 固定文案 + `sufficiency` |

> 「料足」的精神：**有什麼資料就多演一章**。沒下載法人→跳過 8；沒財報→跳過 9；歷史不夠→11 自動降級（沿用情境扇形既有降級）。

## 4. Beat 選取與排序規則

- 章節**固定順序**（§3 表）。
- 每章：資料 `available` 才產生 beat；可用就挑**最顯著**的 1–3 個（例：events 章挑最近、幅度最大的 2–3 個 K）。
- 全段 beat 數 **上限 14**；超過時，保留 `intro / trend / levels / events / watch / outro` 這幾個骨幹章節，其餘依資料顯著度遞減取捨。
- 每個 beat 帶 `confidence`：資料少（如序列短、樣本不足）→ `low`，前端旁白加「資料少，僅供參考」並降視覺權重。

## 5. 預期線（watch level）——唯一接近「未來」的地方，硬規則

- **只能是條件式關卡**，不得是預測路徑。
- 畫法：現價右側**虛線水平線**，標籤 + 條件文字：
  - 「**如果突破** {上方關卡} 且站穩 → 技術派接下來會看上面的 {再上一個關卡}。」
  - 「**如果跌破** {下方關卡} → 會看下面的 {再下一個支撐}。」
  - 「{季線/月線} 目前被當**動態支撐/壓力**在看。」
- 關卡來源：`levels` 波段關卡 + rolling `high_20/60/120`、`low_*` + 動態均線；可加整數關卡。
- **必加**收尾句：「**沒突破/跌破前，都只是區間；突破才算數。**」
- 搭配 `scenario` 章的歷史情境扇形（標非預測）。
- **嚴禁**字眼：會漲到、目標價、會反轉、該買/該賣、勝率、機率、前兆（紅線測試掃）。

## 6. API

- 後端：`app/analyze/chart_tour.py` → `build_chart_tour(payload: dict) -> dict`（純函數，吃既有 `build_stock_payload` 的結果，不另外打 DB／不重算指標）。
- 併入個股 payload：在 `build_stock_payload()` 末端加 `"chart_tour": build_chart_tour(payload_so_far)`；包 `try/except`，失敗回 `{"available": False}`，**不可拖垮個股頁**。
- 不需額外快取（它只是重組既有資料；個股 payload 本身已含結構快取）。

## 7. 前端：播放器（在大型 K 線圖內）

- **入口**：大型圖工具列加按鈕「**讀圖導覽**」（icon + 文字）。只在 `chartLargeMode` 出現。
- **播放**：
  - 進入後顯示一層**導覽 overlay**：底部/側邊一個「旁白泡泡」(headline/why/caution) + 章節抬頭 + 進度點。
  - 每個 beat：①把 `targets` 在圖上高亮（見下）②其餘元素淡化 ③泡泡滑出。
  - **控制**：上一步／下一步／播放暫停（自動前進，每 beat 可讀停留 ~4–6 秒，可調速）／略過整段／ESC 離開。
  - 顏色**禁紅綠**（用中性藍/金/灰，避免被當漲跌訊號）。
- **高亮動作（依 target type）**：
  - `ma` → 該均線加粗描出 + 發光，其餘均線淡化。
  - `level` / `watch_level` → 畫水平線 + 標籤；`origin_date` 在該根 K 點一個圓點。
  - `region` → 半透明色塊框住日期區間。
  - `candles` → 在指定 K 上方掛箭頭/小標。
  - `subplot` → 對應副圖閃爍/外框（若該副圖沒開，導覽期間暫時顯示）。
  - `scenario_cone` → 重用既有情境扇形繪製。
- **動效原則**：用在「描線、泡泡滑出、圓點脈動」幫理解；**不要**全屏亂閃、不要循環。
- **常駐免責**：overlay 頂或底固定「讀圖識讀教學 · 描述現在 · 非預測 · 非投資建議」。
- 程式碼放 `app/ui/static/chart_tour.js`（新檔，避免 `app.js` 再膨脹）或 `app.js` 明確分區。
