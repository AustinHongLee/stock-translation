# 13. Opus 驗收提示：估價防誤導與個股頁語意

你是一位同時具備金融產品經理、散戶投資教育顧問、UX Designer、軟體架構師的資深顧問。

請你不要重寫整個產品藍圖；這次只做「驗收、挑錯、給下一輪優先級」。產品定位是「股票翻譯機」：新手要看得懂，老手要覺得資料誠實。

## 已實作背景

目前已依照 `docs/12-估價適用性與個股頁重整規格.md` 做了第一輪防誤導重整：

- 對不適合估價的股票加入「目前先不估價」路徑。
- 2330 台積電：股利法降權，改看 PE/PB 倍數敏感度。
- 2542 興富發：營建/盈餘認列跳動，改看 PB/資產角度。
- 3576 聯合再生：不估價，改看營收、毛利率、EPS 是否站穩。
- 7722 LINE Pay：新上市/資料短，不估價。
- 股利法畫面不再直接把數字叫作便宜價、合理價、昂貴價；改成「要求殖利率情境」與「高/平均/低殖利率情境」。
- 股利反推價改成「約」的粗略價格，避免假精確。
- PE/PB 由「區間」改成「倍數敏感度 / what-if」，明確說中心情境等於目前價，不代表合理或便宜。
- 雷達中心主榜文案從「撿便宜」改成「低於股利情境排行」。

## 請驗收的核心問題

1. 對完全股票小白來說，現在的第一層是否仍會把估值數字誤認成買點？
2. 「目前先不估價」是否說得夠明確，還是應該更強硬？
3. 「要求殖利率情境」這個命名是否夠白話？有沒有更好的命名？
4. 2330 顯示 PE/PB 敏感度是否仍會讓人以為它是合理價？
5. 2542 只主推 PB/資產角度是否合理？營建股還缺什麼防誤導提示？
6. 3576/7722 不估價時，應該給新手看哪些替代資訊，才不會覺得產品「沒用」？
7. 雷達中心「低於股利情境排行」這個名稱是否仍有暗示買點的風險？
8. 如果要讓股票老手驗收，下一輪最該補的三個資料或公式是什麼？

## 請特別挑錯

- 文字是否還有「合理、便宜、昂貴、低估、高估」的誤導感。
- 是否還有假精確，例如把不確定的估值顯示到小數點。
- 是否把「what-if 敏感度」講成「估價」。
- 是否對成長股、循環股、新上市股、虧損股仍顯得太有把握。
- 是否有資訊架構上的問題：第一層、第二層、第三層是否擺錯權重。

## 請輸出

請用繁體中文回答，格式如下：

1. 最危險的 5 個剩餘誤導風險。
2. 四個案例逐檔批判：2330、2542、3576、7722。
3. 雷達中心下一輪改法。
4. 個股頁 UI 下一輪改法。
5. Codex 可施工的 Phase 2 拆分，每項包含目標、檔案範圍、驗收標準、不做什麼。
