# 15. 旗艦下一步施工指導書（四工作流）

> 這份是**指導書，不是程式碼**。它告訴施工者「做什麼、邊界在哪、怎麼算才對、怎麼驗收」，
> 實際程式由施工方（Codex/工程）照本文與既有模組邊界寫。
> 對齊：`04 技術架構`、`06 風險與過度設計`、`09 旗艦前導書`、`12 估價重整`、`14 Opus 驗收回覆`。

本輪要交付四個工作流：

- **A · 把持倉報酬算對**（含息總報酬 + XIRR + 大盤對比）
- **B · Excel 匯出**
- **C · 強化白話分析**（不接 AI，規則式升級到 6 維度）
- **D · AI 助理**（安全版規格；先交規格與防護，串接 LLM 留最後且需老闆拍板）

---

## 共同紀律（四個工作流都必須遵守）

1. **分析層純函數**：所有算錢、算報酬、算分類的邏輯，固定輸入 → 固定輸出，可單元測試，不碰網路、不碰 AI（延續《04》《06》）。
2. **解說層三鐵則**：解說不自己算數字、不給買賣建議、所有數字可追溯資料源與日期。
3. **防誤導語意**（延續《12》《14》）：不准出現「便宜價／合理價／昂貴價／低估／高估／該買」；估值欄一律講「情境／敏感度／參考」；估算值不顯示過度小數；紅綠只給「漲跌」這種事實，不給「位階」這種判斷。
4. **紅線**：不下單、不串券商交易、不報明牌、不預測股價、不販售或上傳持倉資料。
5. **每個分析模組附契約測試**：放 `tests/`，跟既有 `test_portfolio_calculator.py`、`test_valuation_analysis.py` 同規格。

### 建議施工順序

```
B-v1（Excel，先出安全的快速價值）
   └─ A（報酬算對：含息 + XIRR + 大盤）  ← 最該補的洞，B-v2 依賴它
         └─ B-v2（Excel 補含息/XIRR 欄）
C（強化白話，獨立，可平行）
D（AI 助理：先寫「防護＋去識別化」，串 LLM 最後、需拍板）
```

理由：B-v1 安全又快、先讓人有感；A 是「管得動」這支柱的地基（現在報酬沒算股利，數字偏低會誤導）；C 不依賴別人可平行；D 風險最高，規格先行、實作最後。

## 工作流 A · 把持倉報酬算對（含息總報酬 + XIRR + 大盤對比）

### 目標
讓持倉的「賺賠多少」算到對：把**領到的股利**算進去（含息），用 **XIRR** 反映「錢什麼時候投入」，並跟**大盤**比，回答「我自己選股有沒有贏大盤」。

### 為什麼（痛點 P3）
現況 `app/portfolio/calculator.py` 只算了「市值 − 成本」的未實現／已實現損益，**完全沒算股利**，也沒有年化報酬。對存股族，少算股利會讓報酬系統性偏低，是會誤導的硬傷（《09》支柱二、R10）。

### 檔案範圍
- 新增 `app/portfolio/performance.py`（純函數，對應《09》規劃的 `performance/` 模組）。
- 沿用 `app/portfolio/models.py` 的 `PortfolioTransaction`（已有 trade_date / side / shares / price / fee / tax）。
- 配息來源沿用已同步的官方除息資料（`DividendRecord`，`source == "TWSE_TWT49U"` 那批為實際除息）。
- 大盤序列沿用既有同步機制抓 `0050`（或加權指數）日收盤。
- `app/web/api.py` 增加回傳欄位；前端持倉頁顯示。
- 測試 `tests/test_portfolio_performance.py`。

### 公式與資料模型（要算對的重點）

1. **每檔累計現金股利（含息的核心）**
   重建「持股時間軸」：把交易按日期排好，對每一個**除息日**，算當天持有股數 × 該次每股現金股利 = 該次領到的現金。把整段持有期間每次除息加總 = 累計現金股利。
   - 只用**已同步的官方除息資料**，沒同步到的不臆造（缺就標「配息資料可能不完整」）。
   - 股票股利（配股）要把股數調整算進成本與股數（延續《09》R15）；第一版若先只做現金股利，必須在畫面明講「目前只計現金股利」。

2. **含息總報酬（金額與率）**
   ```
   總報酬金額 = 未實現損益 + 已實現損益 + 累計現金股利
   總報酬率   = 總報酬金額 / 成本基礎
   ```
   這是「累積」口徑，直覺、好懂，當主顯示。

3. **XIRR（年化、考慮投入時點）**
   現金流定義：
   - 買進：現金流出（負）＝ `shares*price + fee`
   - 賣出：現金流入（正）＝ `shares*price − fee − tax`
   - 領息：現金流入（正）＝ 該次現金股利淨額
   - 期末未實現：把「今天的市值」當成一筆**虛擬賣出**現金流入（正），日期＝資料日。
   解 `Σ CF_i / (1+r)^(days_i/365) = 0` 的 r。用二分法或牛頓法；**無解／只有一筆現金流／全同號 → 回 None**，畫面顯示「資料不足以年化」。

4. **大盤對比**
   對照組：把**每一筆實際投入金額**，在當天改成買入 `0050`（同金額），用同一套 XIRR/累積報酬算對照組報酬。並列顯示「你的組合 vs 同期同樣投入買 0050」。標明「對照僅為相對參考，非預測、非建議」。

### 驗收標準
- 有手算範例（2~3 筆交易 + 1 次除息）對得起來，誤差在四捨五入內。
- XIRR 對「單筆買進抱到今天」這種簡單案例，等於用單利反推的合理值；無解時回 None 不丟例外。
- 含息總報酬 ≥ 不含息報酬（同組資料），且差額 = 累計股利。
- 缺配息資料時，畫面明確標「配息資料可能不完整」，數字不靜默偏低。
- 純函數契約測試齊全。

### 不做什麼
- 不做盤中即時跳動（《09》O12，收盤後更新一次即可）。
- 不做多幣別／美股（《09》O15，台股先）。
- 不臆造沒同步到的配息；不接券商對帳單 OCR（《09》O16）。

### 給小白一句話
「以後報酬會把你領到的股利也算進去，還會告訴你『換算成每年賺幾 %』，以及『這樣是不是比直接買 0050 好』——但這是看過去，不是預測未來。」

## 工作流 B · Excel 匯出

### 目標
一鍵把**已經有的資料**倒成 `.xlsx`，讓使用者存檔、備份、或拿去別的地方自己看。安全、好做、馬上有感。

### 為什麼
《09》把「資料匯出」列為 Pro 功能。它只是把現成資料輸出，不產生任何新判斷，是**風險最低、CP 值最高**的一塊。

### 檔案範圍
- `app/web/api.py` 新增匯出端點，例如 `GET /api/export/portfolio.xlsx`、`GET /api/export/stocks/{id}.xlsx`。
- 產生檔案用 `openpyxl`（或本 repo 既有 xlsx 流程）。
- 前端持倉頁與個股頁各加一顆「匯出 Excel」按鈕。

### 內容（分頁設計）
1. **持倉總覽**：代號／名稱／股數／平均成本／成本基礎／最新收盤＋資料日／市值／未實現損益／報酬率／（A 完成後加）含息總報酬／XIRR／佔比。
2. **交易紀錄**：日期／代號／買賣／股數／價格／手續費／稅／備註。
3. **配息紀錄**（A 完成後）：除息日／每股現金股利／當時持股／該次領息。
4. **單檔深入**（個股匯出，選配）：股利歷史、估值「情境」、近幾季財報摘要、白話健檢結論。

### 防誤導要求（沿用《14》）
- 估值欄位標「情境／敏感度」，**絕不**寫「便宜價／合理價」。
- 估算值精度比照前端（價格用「約」、倍數與基數不過度小數）。
- 每個分頁**第一列**固定放：資料日期 + 資料來源 + 一行免責（「本表為已同步資料之整理，非投資建議，不預測股價」）。

### 驗收標準
- 產出的 `.xlsx` 能用 Excel／Numbers／LibreOffice 正常開啟。
- 表內數字與 App 畫面一致（同一筆持倉，市值/報酬對得起來）。
- 欄位有中文表頭與單位；金額千分位；日期格式一致。
- 每分頁有免責列與資料日。

### 不做什麼
- 不做 PDF 匯出（另案）、不做雲端同步。
- 不把任何估值欄寫成可買賣的價格。
- B-v1 可先用現有報酬數字出貨；含息/XIRR 欄等 A 完成再補（B-v2）。

### 給小白一句話
「按一個鍵，就能把你的持倉和交易存成 Excel 檔放在自己電腦，想怎麼看、怎麼備份都行。」

## 工作流 C · 強化白話分析（不接 AI）

### 目標
讓**規則式**健檢讀起來更像人在解釋、更會講「為什麼」和「所以要注意什麼」——但**每一句仍可驗證、不預測、不報明牌**。這是「路 B」：不接 LLM，純升級模板。

### 為什麼
現況 `app/explain/rule_based.py` 是 4 維度（趨勢／位階／獲利／波動）的樣板填空，句子偏「報數字」。《09》支柱一要的是 **6 維度**且更白話。同時要修掉《14》指出的顏色誤導。

### 檔案範圍
- `app/explain/rule_based.py`（擴充維度與模板）。
- 接 `app/analyze/suitability.py`、`app/analyze/valuation.py` 已算好的結論進健檢（解說層只引用、不重算）。
- `app/ui/static/app.css`（位階卡改中性色）。
- 測試 `tests/test_rule_based_explain.py`。

### 具體升級
1. **補到 6 維度**：現有趨勢／位階／獲利／波動 + 新增「**股利穩定性**」「**估值適用性**」，後兩者直接引用 suitability/valuation 的結論翻成白話。
2. **每張卡四段式**：數字 → 白話翻譯 → 為什麼會這樣 → 要注意什麼（仍是模板，不是 AI 生成）。
3. **串連、不孤立**：把相關訊號連起來做**誠實 framing**。例：位階偏低 + 營收連月衰退 → 「現在看起來低，可能是有原因（基本面在走弱），不是單純便宜」——而不是亮綠燈。
4. **修顏色誤導**（《14》風險 1）：價格位階改中性色；紅綠只保留給漲跌。
5. **語氣隨信心調整**：資料品質高/中/低，用詞強弱跟著變（低信心就講得更保守）。

### 驗收標準
- 6 維度齊全，每維度都有「為什麼／要注意」段。
- 固定輸入 → 固定輸出，可單測（不可有隨機或外部呼叫）。
- 位階卡在近一年低位時**不是綠色**；測試補一條斷言。
- 全文掃描無「便宜／合理／該買／會漲」等字。

### 不做什麼
- 不接任何 LLM。
- 不臆測沒有的資料（缺資料就誠實說缺）。
- 不做開放式問答（那是工作流 D）。

### 給小白一句話
「健檢會從『冷冰冰報數字』變成『像有人在旁邊解釋為什麼、要注意什麼』，但它只會講有根據的話，不會幫你預測或叫你買。」

## 工作流 D · AI 助理（安全版規格；串接 LLM 留最後且需拍板）

> 這一塊**先交規格與防護，不先接 LLM**。《09》把 AI 助理排在最後一個里程碑 M3，且列為最高風險（R11）。本工作流的交付物是「規格 + 三道防線 + 去識別化」三件**可單測**的東西；真的串 LLM 寫對話，列為最後一步，需老闆明確拍板。

### 定位（嚴格封閉，這是重點）
**不是開放聊天機器人**（《09》O14）。助理只准做兩件事：
1. **解讀「你自己的本機資料」**：把已經算好的持倉／某檔指標的結論，講成個人化白話摘要。
2. **當導航**：「你想看懂哪個面向？」→ 帶你去對應頁面或教學。

LLM 的角色是「**把已知結論講白**」，**不負責算數、不負責判斷買賣**。所有數字、分類、報酬都來自分析層純函數，LLM 只拿成品。

### 檔案範圍
- 新增 `app/assistant/`（L3，對應《09》`assistant/`）。
- 子件：`guardrail.py`（禁止字眼過濾，**先寫先測**）、`deidentify.py`（去識別化）、`context_builder.py`（只組裝已驗證結論）、`fallback.py`（退回規則式）。
- 測試 `tests/test_assistant_guardrail.py`、`tests/test_assistant_deidentify.py`。

### 三道防線（對齊《04》三鐵則 + 《09》R11）
1. **輸入端**：只餵「已驗證的結構化結論」，不接受「請預測 / 請推薦」這類任務型輸入。
2. **Prompt 端**：系統提示明令——不得輸出買進／賣出／目標價／漲跌預測；只能解讀與導航；固定附免責。
3. **輸出端：禁止字眼過濾器**（買進、賣出、目標價、會漲、會跌、建議買/賣、加碼、抄底…）。命中就攔截，改走安全模板或要求重生。**這個過濾器必須在接任何 LLM 之前就寫好、測好。**

### 隱私（《09》R9 第一級紅線）
- 送 LLM 前**去識別化**：拿掉可識別個資，金額可選擇模糊化（例如只送比例不送絕對金額）。
- 本地優先；提供一鍵清除對話與資料。

### 降級（不可開天窗）
- 無網路 / 額度用完 / LLM 失敗 → **退回規則式**（工作流 C 的輸出），畫面不可空白或報錯。

### 驗收標準
- 過濾器單測：準備一批「越線句子」（含買賣/目標價/漲跌預測），**全部被攔下**。
- 去識別化單測：輸出不含可識別個資；金額模糊化可開關。
- 斷網模擬：助理自動退回規則式，不丟例外。
- 全程不出現買賣／目標價／漲跌預測字樣。

### 不做什麼（現在）
- 不先接真 LLM 寫自由對話。
- 不做「分析任何一檔股票」的開放輸入框。
- 先交付「規格 + 過濾器 + 去識別化 + 降級」四件可單測的東西；LLM 串接是最後一步、需拍板。

### 給小白一句話
「未來如果做 AI 助理，它只會解讀『你自己的持倉』和帶你看對應頁面，**永遠不會**叫你買或賣、也不會說會漲會跌——而且這是最後才做、要老闆同意才做的部分。」

## 一句話總結

這四件事的順序就是它的價值排序：**先把報酬算對（A）、先給安全的匯出（B）**，是地基和快速價值；**把白話講得更好但不冒險（C）**，是體驗；**AI 助理（D）規格先行、實作最後**，因為它最容易把前面所有「誠實」一次毀掉。每一步都守同一條線——這個產品賣的是「看懂與安心」，不是「明牌」。
