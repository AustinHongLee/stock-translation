## 2. 建議設計語言：「沉穩翻譯台」

延續現有海軍藍主色，強化「金融可信 + 白話親和」雙特質，避免券商 App 的霓虹感或記帳 App 的幼稚感。

### 2.1 Design Tokens（建議寫入 `app.css` `:root`）

```css
:root {
  /* 品牌 */
  --brand-primary: #1c3d5a;      /* 深海藍 — 導航、主按鈕 */
  --brand-primary-hover: #132b40;
  --brand-accent: #2b6cb0;       /* 資訊強調 — 連結、圖表線 */

  /* 語意色（台股） */
  --tone-up: #e03131;            /* 漲 — 紅 */
  --tone-down: #2f9e44;          /* 跌 — 綠 */
  --tone-flat: #6b7785;
  --tone-caution: #f08c00;
  --tone-positive: #2f9e44;      /* 健檢「良好」用綠，非漲跌 */
  --tone-unknown: #9aa6b2;

  /* 表面 */
  --bg-canvas: #edf0f4;
  --bg-surface: #ffffff;
  --bg-surface-muted: #f6f8fa;
  --border-default: #e3e8ef;
  --border-strong: #cfd6df;

  /* 字體 */
  --font-sans: "Noto Sans TC", "Microsoft JhengHei", "Segoe UI", sans-serif;
  --font-mono: "JetBrains Mono", "Cascadia Mono", monospace; /* 股價數字 */

  /* 字級 */
  --text-hero: 30px;    /* 總資產 */
  --text-kpi: 24px;     /* 報價、指標 */
  --text-title: 18px;
  --text-body: 15px;
  --text-caption: 12px;

  /* 圓角與陰影 */
  --radius-sm: 8px;
  --radius-md: 12px;
  --radius-pill: 999px;
  --shadow-card: 0 2px 12px rgba(28, 61, 90, 0.08);
  --shadow-float: 0 14px 36px rgba(28, 61, 90, 0.16);

  /* 間距（8px 基數） */
  --space-1: 4px;
  --space-2: 8px;
  --space-3: 12px;
  --space-4: 16px;
  --space-5: 24px;
}
```

### 2.2 字體策略

| 用途 | 字體 | 理由 |
|------|------|------|
| 內文、標題 | Noto Sans TC | 跨平台一致、免費、繁中優化 |
| 股價、損益、% | 等寬或 tabular nums | 數字對齊，專業感 |
| 數字 OpenType | `font-variant-numeric: tabular-nums` | 不需額外字體檔 |

```css
.price, .kpi-value, td:nth-child(n+3) {
  font-variant-numeric: tabular-nums;
  letter-spacing: 0.02em;
}
```

### 2.3 色彩使用規則

1. **主色只用在導航與主 CTA**，避免整頁藍色過重  
2. **漲跌色僅用於數字**，不用於背景大色塊（降低焦慮）  
3. **面板左色條**區分模組（現有做法很好，建議標準化）：

| 模組 | 左色條 | 含義 |
|------|--------|------|
| 盤中報價 | `--brand-accent` 藍 | 即時 |
| 每月營收 | `--tone-positive` 綠 | 成長 |
| 基本面 | `--tone-caution` 琥珀 | 財報 |
| 健檢警示 | `--tone-caution` | 注意 |
| 持倉解讀 | `--brand-primary` | 個人 |

## 3. 版面與資訊架構

### 3.1 建議桌面版骨架（對齊 mockup `01-儀表板.svg`）

```
┌─────────────────────────────────────────────────────────────┐
│ [股] 投資作業系統    [首頁|持倉|個股]    🔍 搜尋    [新手|進階] 🙈 │
├──────────┬──────────────────────────────────────────────────┤
│ 自選股   │  主內容區（max-width: 1200px）                    │
│ (240px)  │  · 首頁：總資產 + 警示 + 晨報 + 雙欄列表          │
│          │  · 持倉：KPI + Tab + 表格 + 白話解讀              │
│          │  · 個股：三層（30秒看懂 → 健檢卡 → 原始表）        │
└──────────┴──────────────────────────────────────────────────┘
```

**具體 CSS 調整：**

```css
@media (min-width: 1024px) {
  .workspace {
    flex-direction: row;
  }
  .sidebar {
    display: flex;
    flex-direction: column;
    width: 240px;
    padding: var(--space-4);
    border-right: 1px solid var(--border-default);
    background: var(--bg-surface);
  }
  .main-pane {
    flex: 1;
    max-width: none;
  }
}
```

### 3.2 個股頁「三層深度」（對齊 mockup `03-個股三層頁.svg`）

| 層級 | 內容 | 新手模式 | 進階模式 |
|------|------|----------|----------|
| L1 30 秒 | 明牌驗證工作台 3 句 | 永遠展開 | 永遠展開 |
| L2 健檢 | 報價 + 四維度卡片 | 顯示白話摘要 | 顯示完整 bullet |
| L3 原始 | 營收/財報/股利表、日線表 | **預設摺疊** | 預設展開 |

**實作方式：** 在 `.revenue-panel`、`.financial-panel`、`.table-panel` 加 `<details>` 或自訂 accordion，由 `body[data-density="beginner"]` 控制預設狀態。

### 3.3 首頁儀表板強化

對照 mockup，建議調整優先順序：

1. **總資產卡**：數字加大（已有 30px），補「今日損益」次行 + 小型 sparkline  
2. **警示卡**：用 icon + 色點區分類型（除息 / 技術 / 資料缺口）  
3. **晨報卡**：左側色條 + 底部固定 disclaimer（已有，保持）  
4. **雙欄列表**：持倉顯示「報酬率 + 佔比」，自選顯示「今日漲跌」

## 4. 元件規範

### 4.1 按鈕層級

| 類型 | 樣式 | 使用場景 |
|------|------|----------|
| Primary | 實心 `--brand-primary` | 搜尋、儲存交易、加入自選 |
| Brand CTA | 實心 + 圖示 | 「我買了」（已有 `.brand-button`） |
| Secondary | 白底描邊 | 同步、清空、帳戶選擇 |
| Text | 無框藍字 | 「看全部」「看持倉」 |
| Danger | 紅框 | 刪除交易（已有 `.danger-button`） |

**建議：** Primary 全頁最多 1 個「最突出」按鈕，避免個股頁同時有 3 個 primary（目前「加入自選」「我買了」都是 primary）。

```css
/* 建議：watchlist 改 secondary，只保留「我買了」為 primary */
#watchlistButton { /* 改為 secondary-button 視覺 */ }
```

### 4.2 表格

- 持倉表：第一欄左對齊，數字右對齊（已有）  
- 損益欄：加上 `toneClass` 著色（部分已有，持倉表可統一）  
- 佔比欄：保留 `.allocation-track` 進度條（已有，很好）  
- 斑馬紋：可選 `tbody tr:nth-child(even)` 淺灰，提升長表可讀性

### 4.3 載入與空狀態

目前同步時只有 `setSyncing` 改按鈕文字。建議：

```css
.skeleton {
  background: linear-gradient(90deg, #f0f0f0 25%, #e8e8e8 50%, #f0f0f0 75%);
  background-size: 200% 100%;
  animation: shimmer 1.2s infinite;
  border-radius: var(--radius-sm);
}
```

用於：報價四格、健檢四卡、首頁總資產。

### 4.4 名詞解釋 Popover

現有 overlay 設計良好。建議小改：

- 加「相關名詞」連結（同 glossary 別名）  
- 鍵盤 `Tab` 可聚焦關閉鈕（a11y）  
- 圓角從 8px 改 12px 與全站一致

## 5. 互動與微動效

遵循《09》「踏實感而非刺激感」：

| 允許 | 避免 |
|------|------|
| 按鈕 hover 100ms 色彩過渡 | 股價閃爍動畫 |
| 頁面切換 fade 150ms | 數字滾動 ticker |
| 圖表 crosshair tooltip | 全螢幕紅綠呼吸燈 |
| 同步中 spinner | 推播式彈窗 |

```css
.primary-button,
.secondary-button,
.stock-item {
  transition: background-color 0.15s ease, border-color 0.15s ease, color 0.15s ease;
}
```

---

## 6. 無障礙（a11y）

現有 `sr-only`、`aria-label` 基礎良好。建議補：

1. **色覺友善**：漲跌除顏色外加 ▲▼ 符號（部分已有 `formatSignedNumber`）  
2. **對比度**：`--muted` #6b7785 對白底約 4.6:1，caption 可加深至 #5a6672  
3. **焦點環**：`.term-link:focus` 已有，擴展到所有 `button:focus-visible`  
4. **Sheet 切換**：`aria-selected` 綁定 nav button

---

## 7. 深色模式（P2，可選）

本地工具使用者可能夜間查看。建議用 `prefers-color-scheme` 預留：

```css
@media (prefers-color-scheme: dark) {
  :root {
    color-scheme: dark;
    --bg-canvas: #12171d;
    --bg-surface: #1a2129;
    --text: #e8edf2;
    /* 漲跌色維持紅漲綠跌，僅降低飽和度 */
    --tone-up: #ff6b6b;
    --tone-down: #51cf66;
  }
}
```

首發可不做，但 tokens 架構應先到位。

## 10. 一句話風格定位

> **看起來像銀行對帳單一樣可信，讀起來像朋友解釋一樣清楚。**

這應作為所有 UI 決策的北極星：當「炫」與「懂」衝突時，永遠選「懂」。
