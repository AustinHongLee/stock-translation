## 3. 覆蓋缺口分析

以下模組**有實作但缺少直接測試**，建議補上：

| 模組 | 風險 | 建議測試 |
|------|------|----------|
| `app/analyze/financial.py` | 中 | EPS/ROE/ROA 邊界（除零、負值、缺欄位） |
| `app/web/server.py` | **高** | GET/POST/PUT/DELETE 路由、錯誤碼、靜態檔安全 |
| `app/quote/providers.py` | 中 | MIS 報價 timeout、缺成交價 fallback |
| `app/cli.py` | 低 | CLI 子命令 smoke test |
| `app/ui/static/app.js` | **高** | 目前零前端測試（建議 Vitest 或 Playwright） |

### 3.1 `build_stock_payload` 已驗但可再加強

現有測試已覆蓋主要 happy path，建議補：

- 無本地價格時的 `summary` / `quote` 降級行為
- `historical_yield` 資料不足時的 warning 文案
- 空 watchlist / 空 portfolio 的 dashboard 組合

### 3.2 持倉驗證邏輯

`server._validate_portfolio_state` 在 HTTP 層執行，但**沒有整合測試**驗證：

- 賣超 → 400 Bad Request
- 更新不存在交易 → 404
- 手續費/稅計入成本（專家驗收題目）

## 4. 建議測試基礎建設

### 4.1 短期（標準庫即可）

```python
# tests/test_server_http.py — 使用 http.client 或 urllib 打本機 handler
# 不需額外依賴，符合 pyproject「零依賴」原則
```

### 4.2 中期

在 `pyproject.toml` 新增 optional dev 群組：

```toml
[project.optional-dependencies]
dev = ["coverage>=7.0"]
```

並加入：

```powershell
python -m coverage run -m unittest discover
python -m coverage report -m
```

目標：核心模組（`portfolio`、`analyze`、`explain`、`sync`）行覆蓋率 **≥ 85%**。

### 4.3 長期

- **E2E**：Playwright 驗證「搜尋 → 同步 → 看健檢 → 加自選 → 記交易」主路徑
- **契約測試**：為 `/api/stocks/{id}` JSON schema 建立 snapshot，防止 UI 與 API 漂移
