# 結構指紋

「結構指紋」是個股頁頂部摘要卡，用來描述一檔股票近期價格序列的結構性格。它只回答「目前的價格路徑長得像什麼」，不回答後續方向，也不提供操作指令。

## 位置與資料來源

- 顯示位置：個股頁頂部摘要區，和股價、資料日期、體質燈號同層。
- 計算視窗：預設顯示 250 個交易日；內部會保留足夠樣本給頻譜斜率估計。
- 快取：API 使用 `structure::<stock_id>::<last_close_date>::250` 作為 `app_cache` key。新收盤日進來會自然換 key。
- 批次限制：`/api/local-data` 不計算結構指紋，避免全市場盤點時逐檔重算。

## 六個維度

| 維度 | 數值來源 | 讀法 |
|---|---|---|
| 延續性 | DFA-1 Hurst | 描述報酬序列的自相關結構，不含方向。 |
| 複雜度 | Permutation Entropy | 描述順序型態是否接近隨機或較格式化。 |
| 波動聚集 | 絕對報酬自相關 | 描述大波動是否容易成群出現。 |
| 噪音色 | log price spectral slope | 描述價格路徑比較破碎、平衡或平滑。 |
| 湍流程度 | 20 日已實現波動百分位 | 描述近期波動在自身歷史中的相對位置。 |
| 同步性 | Phase 2 市場資料 | 個股頁鎖定，需跨股資料才開放。 |

## 信心與降級

- `high`：樣本與估計品質足夠，顯示實色格。
- `medium`：顯示半透明，標成僅供參考。
- `low` / `insufficient`：顯示灰色虛線，只說資料不足，不顯示精確小數。
- `locked`：同步性需要市場層級資料，個股頁只顯示鎖定狀態。

## 安全紅線

結構指紋不得使用方向承諾、操作語言、勝負率語言或目標數字語言。每個維度都必須帶 `forbidden` 文字，提醒使用者「這是描述，不是買賣指令」。新增文案或報告輸出時，請先跑：

```bash
python -m pytest tests/test_structure_metrics.py tests/test_structure_registry.py tests/test_structure_guardrails.py -q
```

## 延伸方向

本輪不做 SampEn 卡片、MFDFA、全市場排行，也不做市場心智雷達。未來若要開同步性，前置條件是跨股報酬矩陣與市場層級資料載入器。
