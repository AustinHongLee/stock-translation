# 01. 使用者研究:散戶日常、痛點與真正有價值的功能

> 本文回答三個問題:散戶每天在做什麼(A)、卡在哪裡(B)、什麼功能真的有用(C)。
> 所有後續設計決策都必須能追溯回本文的某個痛點,否則就是過度設計。

---

## A. 一般散戶每天最常做的事

「散戶」= 用自己的錢、自己做決定的個人投資者。對照組是「法人」(銀行、基金公司等專業機構)。

依時間軸整理台灣散戶的典型一天:

| 時段 | 行為 | 頻率 |
|------|------|------|
| 早上(開盤前) | 滑手機看美股昨晚漲跌、看財經新聞標題 | 每天 |
| 09:00–13:30(盤中) | 上班偷看持股漲跌,反覆刷新報價 | 每天、一天數十次 |
| 盤中 | 看到自己的股票大跌,慌張地想找原因 | 每週數次 |
| 收盤後 | 看今天賺賠多少 | 每天 |
| 晚上 | 從 LINE 群、PTT、Dcard、YouTube 聽到「明牌」(別人推薦的股票),去查這支能不能買 | 每週數次 |
| 晚上 | 查個股:看線圖、看新聞、看別人怎麼說 | 每週數次 |
| 不定期 | 猶豫該不該賣:賠錢想凹回本、賺錢怕跑太早 | 持有期間反覆發生 |
| 幾乎不做 | 記錄自己為什麼買、為什麼賣 | 極少 |

兩個關鍵觀察:

1. **散戶的起點通常是「聽到一支股票」,而不是「主動篩選市場」。** 流程是:聽到明牌 → 想驗證 → 不會驗證 → 憑感覺買。產品最大的切入點就在「想驗證但不會驗證」這一步。
2. **散戶花最多時間的行為(盤中刷報價)其實沒有產生任何決策價值,純粹是焦慮。** 產品不該強化這個行為(例如推播即時漲跌),而該把注意力導向「看懂」。

## B. 散戶最常遇到的痛點

按嚴重度排序:

| # | 痛點 | 說明 | 嚴重度 |
|---|------|------|--------|
| P1 | **看不懂專業介面** | 打開券商 App 或看盤軟體,滿屏的 K線、RSI、MACD(各種技術圖表與指標,見《名詞解釋》),沒有任何說明。軟體假設你已經懂了 | ★★★★★ |
| P2 | **無法判斷一支股票「好不好、貴不貴」** | 聽到明牌後沒有驗證能力,只能看「最近有沒有漲」來決定,等於追高 | ★★★★★ |
| P3 | **買了之後不知道何時該賣** | 沒有出場依據 → 賠錢凹單(虧損持續擴大也不賣)、賺錢太早跑 | ★★★★☆ |
| P4 | **看新聞無法判斷利多還是利空** | 「營收創新高」股價卻跌;新聞有時是大戶出貨的工具。散戶無法分辨 | ★★★★☆ |
| P5 | **資訊分散** | 看線圖一個 App、看新聞一個網站、看討論又一個論壇,拼湊成本高 | ★★★☆☆ |
| P6 | **沒有紀錄,無法從錯誤學習** | 不記得自己當初為什麼買,同樣的錯誤重複犯 | ★★★☆☆ |
| P7 | **情緒化交易** | 大跌恐慌賣出、大漲衝動追入 | ★★★☆☆(難以用軟體直接解決) |

> P7 的誠實註記:情緒問題無法靠工具根治,但 P2、P3、P6 解決後,情緒化交易會間接減少——有依據的人比較不慌。

## C. 真正有價值的功能

核心洞察:**散戶缺的不是「更多資料」,而是「幫我看懂這些資料,告訴我這代表什麼」。**
市面上的看盤軟體在比資料多、圖表專業;本產品反過來:資料可以少,但每一筆都附白話解讀。

### 有價值(直接對應痛點)

| 功能 | 解決的痛點 | 價值說明 |
|------|-----------|---------|
| **個股健檢報告**:輸入代號,產出一頁白話摘要(這家公司賺不賺錢?股價在歷史上算貴還便宜?最近趨勢如何?有什麼風險?) | P1, P2 | 本產品的差異化核心。把「驗證明牌」從不可能變成 30 秒 |
| **AI 白話解說**:所有指標附「這個數字代表什麼、一般怎麼解讀」 | P1 | 邊用邊學,不需要先上課 |
| **名詞即點即解**:介面上任何專業名詞,點一下就有白話解釋 | P1 | 消除門檻的基礎建設 |
| **自選股清單 + 每日異動摘要**:「你關注的台積電今天 -3%,可能與 ___ 有關」 | P5 | 把分散的資訊聚合成一頁 |
| **交易筆記**:買入時強制(輕量地)記一句理由,賣出時回顧 | P3, P6 | 建立決策依據的習慣 |
| **新聞 AI 摘要與歸類**:這則新聞在講什麼、屬於公司營運/產業/市場情緒哪一類 | P4 | 降低被新聞帶風向的機率 |

### 看似有價值、實際上不該做

| 功能 | 為什麼不做 |
|------|-----------|
| 預測股價/AI 報明牌 | 做不到也不該做。預測錯誤會摧毀信任,且在台灣可能涉及投顧法規(見《06-風險》) |
| 即時報價、盤中秒級更新 | 強化焦慮行為(見 A 的觀察 2),且即時行情資料授權昂貴 |
| 全套專業技術指標 | 目標用戶看不懂,做了等於把產品變回他們用不來的看盤軟體 |
| 社群討論/聊天室 | 明牌文化的溫床,與產品「幫你自己看懂」的定位矛盾 |

---

## 本文結論(一句話)

**產品定位:給看不懂看盤軟體的人的「股票翻譯機」——把市場資料翻譯成人話,讓散戶第一次有能力自己驗證一支股票。**
