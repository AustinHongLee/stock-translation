# 圖表視覺優先原則（Visual-First Principles）

> 跨功能硬規則，**適用於**：結構指紋卡（HANDOFF_3）、讀圖導覽（HANDOFF_5）、關係層（HANDOFF_6）、以及之後所有圖上解說。
> 由 Claude 整理。未 commit/未 push。理由底線不變：**描述、非預測**；視覺也一樣，不得暗示方向。

---

## 原則 1：文字長在「圖上」，不丟角落

- 所有解說用 **錨定泡泡（anchored callout）**：一個小泡泡 + 一條**指向線（leader line）**指到對應的 K 棒 / 均線 / 區域 / 交叉點。
- **泡泡自動避讓**：放在目標附近「最空」的方位（上/下/左/右擇一），**不可長期遮住最近幾根 K 棒**。
- **禁止**把主要解說固定塞在右下角 / 側欄當「字幕」。右側 / 角落只留**控制項**（上一步、下一步、暫停、進度點），不承載主要文字。
- 關卡線、收盤點、交叉點的標籤**直接貼在線端 / 點旁**，不另開文字區。

## 原則 2：能用「圖」就不用「字」（visual-first）

每個概念**先給一眼看懂的視覺**，文字退為輔助 / 點開才看。對照表（Codex 照做）：

| 概念 | 視覺編碼（主） | 文字（退為輔助/可展開） |
|---|---|---|
| 跨源確認（價/量/法人 扎不扎實） | **三個方向箭頭並排** ↑↑↑（同向=扎實）/ ↑↓· （背離）+ 同向打勾 | 「這次上漲有量、大戶也買」 |
| 衍生鏈（誰算出誰） | 圖上**連接線/箭頭**：均線 → MACD、均線±波動 → 布林 | 「MACD＝兩條均線的距離」 |
| 階段（糾結→發散 / 收斂→擴張） | **小漏斗/收口 glyph** + 在均線群畫出收斂帶 | 「三條線黏在一起」 |
| 可信度（結構指紋當濾鏡） | **清晰 vs 起霧**：可信低 → 該段半透明/霧化/虛線；可信高 → 清晰銳利 | 「這檔最近很亂，判讀參考就好」 |
| 動能（力氣/過熱） | **溫度計 / 油表 小圖**（中性色） | （數字 KD/RSI 只在細節層） |
| 量能 | **量柱本身上色/高亮**，不另寫「量增」 | hover 才出數字 |
| 支撐/壓力 | 水平線 + **地板/天花板 icon** | 線端標價 |
| 歷史情境 | **扇形**（已是圖） | 「過去類似情況的範圍·非預測」 |
| 趨勢方向 | 均線**描出動畫** + 箭頭沿線 | 「短期比中期熱」 |

- **數字與術語只活在點開的「細節層」**（沿用 HANDOFF_5 §1.5 三層；Layer 0 圖示+一句、零數字零術語）。

## 原則 3：視覺語彙統一

- callout、leader line、箭頭、燈號、glyph、霧化 → 做成**一套共用前端元件**，三個功能共用。
- **顏色禁用紅綠當方向**（沿用既有規則）；用中性色 + **形狀/圖示**區分意義（箭頭方向、打勾、漏斗、溫度計…）。

## 原則 4：一次只強調一個重點，不要擠

- 導覽**逐步**強調；關係層預設只亮主角「**扎不扎實**」，其餘收合。
- 重疊 / 小螢幕時泡泡自動讓位、可關閉；同時最多 1 個主泡泡 + 少量端點標籤。

## 原則 5：每個視覺元素可點 → 展開白話三層

- 點任一箭頭/線/glyph → 出 Layer 0（一句白話）→ 可再點「為什麼/看數字」。
- 對應 HANDOFF_5 的懸停辨識：滑過去先一行「我是誰」。

---

## 視覺不得暗示預測（紅線延伸）

- 箭頭代表「**描述同不同向 / 已發生**」，**不是**「會往哪走」。
- 預期線一律條件式 + 虛線 + 「非預測」標註（沿用 HANDOFF_5 §5）。
- 霧化/清晰代表「**多可信**」，不是「會漲會跌」。

---

## 驗收（給 Codex）

- [ ] 導覽 / 關係 / 結構解說的**主要文字以 anchored callout 呈現**（有 leader line 指向目標）；右下角/側欄不得承載主要解說（只放控制項）。
- [ ] callout 自動避讓，**不長期遮住最近 K 棒**（截圖 smoke 檢查）。
- [ ] 每個關係項/導覽 beat **至少有一個圖示或視覺編碼**，不是純文字。
- [ ] 可信度低的區段有**視覺降級**（霧化/虛線）。
- [ ] 顏色不以紅綠表方向；數字只在點開的細節層。
