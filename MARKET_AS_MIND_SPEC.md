## 0. 先解決你提到的「自相矛盾」

你的第 2 節用「需要什麼資料」分級、第 7 節用「證據成熟度」分級——這是**兩條不同的軸**，混在一起會打架。本規格統一成兩個**正交標籤**，每個指標都同時標：

- **資料軸（Data Tier）**：`D0`=只要 OHLCV｜`D1`=需要跨股票/類股/ETF 橫截面｜`D2`=需要敘事層(新聞/社群/籌碼/Tick/Order Book)。
- **成熟度軸（Maturity Tier）**：`A`=可直接實作(成熟、可重現、可解釋)｜`B`=可研究(有文獻、工程可做、解釋成本高)｜`C`=需代理變數(OHLCV 不足、要建 proxy)｜`D`=理論探索(目前缺可靠觀測量)。

> **MVP = D0 ∩ A**。Phase 2 = D1。Research = D2 或 Tier C/D。
> 分類準則一律用你指定的四問：**有沒有可觀測量／可不可驗證／可不可重現／可不可被推翻**——不用「真假」「玄學」當標準。

## 1. 系統總覽

**定位**：在既有個股頁與報告上，加一層「結構描述」。回答的不是「明天會漲嗎」，而是「現在這檔/這個市場，比較像：湍流？趨勢？白噪音？臨界？高同步？敘事分裂？長記憶？高複雜度？」

**設計理念**
1. 描述而非預測：所有輸出都是「現在的狀態描述」，附資料充足度與信心。
2. 重用優先：建立單一 `app/analyze/structure_metrics.py` 引擎，與既有指標引擎(`indicators.py`)共用 OHLCV，避免再次三套實作。
3. 誠實即差異化：資料不足就標不足、估計不穩就降信心；不可被當成訊號。
4. 漸進：D0→D1→D2、A→B→C→D，一層層加，每層可獨立出貨。

**使用情境**
- 散戶：看「結構指紋卡」一眼理解「這檔現在性格如何」(記憶長不長、亂不亂、是否接近劇變)。
- 進階：在「市場心智雷達」看整體同步度/熵/雪崩風險，判斷「現在是不是大家變成同一顆腦」。
- 研究：實驗室分頁跑 TDA/RQA/網路指標，產生研究報告。

**使用者價值**：把「看不懂的隨機波動」翻成「可描述的結構語言」，降低資訊焦慮；明確不承諾報酬，建立信任(尤其對投資人)。

## 3. 指標完整規格

> 共通慣例：以 **log return** `r_t = ln(P_t/P_{t-1})` 為主要輸入（除非註明）。`N`=樣本數。所有指標輸出**滾動序列**(用視窗 W)+**最新快照**。資料不足 → 回 `null` 並標「資料不足」。

### 3.1 DFA / Hurst 指數 H 〔記憶性〕— D0 / A
- **理論**：分形、長程相關。
- **數學定義**：
  1. 對輸入序列去均值後累加成 profile：`Y(i)=Σ_{k≤i}(r_k - r̄)`。
  2. 切成長度 `n` 的不重疊視窗，每窗最小平方擬合局部線性趨勢並去除，算去趨勢殘差均方根 `F(n)=sqrt( (1/N)Σ (Y - 局部擬合)^2 )`。
  3. 對一系列 `n`（建議 `n ∈ [8, N/4]`，對數等距 ~15 點）做 `log F(n) vs log n` 線性回歸，斜率即 DFA 指數 α ≈ H。
- **變數**：α(=H)；窗階 n 範圍；擬合階(DFA-1)。
- **最小資料**：N ≥ 120(可算)、≥ 250 較穩。
- **計算流程**：profile → 多尺度去趨勢 RMS → log-log 回歸取斜率 + R²(擬合品質)。
- **成本**：低。**穩定度**：高。
- **失真風險**：序列太短(<120)、強趨勢污染(用 returns 而非 price 可緩解)、結構轉折期跨窗失真。回報 R² 作為可信度。
- **白話解讀**：`H>0.55` → 傾向延續近期行為(長記憶)；`H≈0.5` → 接近隨機；`H<0.45` → 傾向反覆/回擺。
- **禁止解讀**：不得解讀為「會繼續漲/一定反轉」；H 是描述**自相關結構**，不是方向。

### 3.2 Multifractal Width Δα (MFDFA) 〔湍流程度〕— D0 / A→B
- **理論**：多重分形(湍流級聯類比)。
- **數學定義**：DFA 推廣到 q 階：`F_q(n) = { (1/Ns)Σ [F²(窗)]^{q/2} }^{1/q} ~ n^{h(q)}`；質量指數 `τ(q)=q·h(q)-1`；用 Legendre 變換 `α=dτ/dq`、`f(α)=qα-τ(q)`。**寬度 Δα = α_max - α_min**。
- **最小資料**：N ≥ 500，建議 ≥ 1000；q ∈ [-4,4]。
- **成本**：中。**穩定度**：中(對 N 敏感)。
- **失真風險**：短序列嚴重高估寬度；極端值主導。N<500 一律不顯示。
- **白話**：Δα 大 → 結構「像湍流」(多種尺度行為混雜、劇烈)；Δα 小 → 行為單一、接近單分形/隨機。
- **禁止**：不得當「波動會放大」的預告。

### 3.3 Permutation Entropy (PE) 〔複雜度〕— D0 / A
- **理論**：資訊熵 / 序數模式。
- **數學定義**：嵌入維度 `d`(3–6)、延遲 `τ`(1)。對每個長度 d 的子序列求其大小排序的序數型態(共 d! 種)；統計各型態頻率 `p_π`；`PE = -Σ p_π ln p_π`，正規化 `PE_norm = PE / ln(d!)` ∈[0,1]。
- **最小資料**：N ≥ 100(且需 ≫ d! 才穩，d=4→24 種)。
- **成本**：低。**穩定度**：高(對雜訊/離群穩健)。
- **失真風險**：d 太大 + N 太小 → 型態取樣不足；平盤(等值)需設 tie 規則。
- **白話**：接近 1 → 高複雜度/接近隨機；偏低 → 有可辨識的順序結構。
- **禁止**：低 PE ≠「可預測會賺」。

### 3.4 Sample Entropy (SampEn) 〔複雜度/規律性〕— D0 / A
- **數學定義**：`SampEn(m,r,N) = -ln(A/B)`；B=長度 m 的模板對中距離< r 的數量、A=長度 m+1 的數量；建議 `m=2`、`r=0.2·std(r_t)`。
- **最小資料**：N ≥ 200。**成本**：中(O(N²)，可用 KD-tree 降到近似 O(N log N))。**穩定度**：中。
- **失真風險**：r 選擇敏感；非平穩期偏移。建議 r 用滾動 std。
- **白話**：高 SampEn → 不規律/難壓縮；低 → 規律、重複性高。
- **禁止**：不得當訊號；與 PE 互為佐證(兩者皆指「複雜度」維度)。

### 3.5 Spectral Slope β (1/f 顏色) 〔噪音性〕— D0 / A
- **理論**：傅立葉 / 臨界(1/f)。
- **數學定義**：對 log 價格(或 returns)做 Welch PSD `S(f)`；在中頻帶(去掉最低與最高各 ~10%)擬合 `log S(f) = -β log f + c`，斜率即 β。
- **最小資料**：N ≥ 256。**成本**：低。**穩定度**：中。
- **失真風險**：頻帶選擇、視窗洩漏；對 returns β≈0(白)，對 log 價格 β≈2(布朗)。**規格固定用 log 價格**並以「離 1/f 有多近」描述。
- **白話**：β 接近 1 → 粉紅噪音(秩序與隨機之間，常見於臨界系統)；接近 2 → 平滑/趨勢化；接近 0 → 白噪音/破碎。
- **禁止**：不得當週期或轉折預告。

### 3.6 Volatility Clustering Index 〔波動聚集/長記憶〕— D0 / A
- **數學定義**：對 `|r_t|`(或 `r_t²`)算自相關 `ρ_k`，k=1..K(K≈20)；指數 = `Σ_{k=1}^{K} max(ρ_k,0)` 或顯著(>2/√N)落後數。亦可擬合 ρ_k ~ k^{-γ} 取長記憶指數。
- **最小資料**：N ≥ 150。**成本**：低。**穩定度**：高。
- **白話**：高 → 「大波之後容易接大波」(波動有記憶)；低 → 波動較無記憶。
- **禁止**：不得解讀為「即將大漲大跌」，只是波動的**聚集性**。

### 3.7 Realized Volatility / Turbulence 百分位 〔湍流(快)〕— D0 / A
- **數學定義**：滾動已實現波動 `RV = std(r_{t-W+1..t})·√252`；輸出其在自身歷史的**百分位**(0–100)。Turbulence 亦可用 Mahalanobis(需多變量→D1)。單股版用 RV 百分位。
- **最小資料**：N ≥ 60。**成本**：低。**穩定度**：高。
- **白話**：百分位高 → 目前波動處於歷史高檔(較「亂」)。
- **禁止**：高波動 ≠ 會跌。

### 3.8 Return Autocorrelation（短期）— D0 / A
- **數學定義**：`ρ_1, ρ_5`(returns 一階、五階自相關) + 顯著性。
- **白話**：`ρ_1>0` 短期動量傾向、`<0` 短期反轉傾向。**禁止**：不得當進出場。

### 3.9 Tail Index (Hill) 〔肥尾/臨界〕— D0 / B
- **數學定義**：對 |returns| 取最大 k 個，Hill 估計 `ξ = (1/k)Σ ln(x_(i)/x_(k+1))`，尾指數 α_tail=1/ξ。
- **最小資料**：N ≥ 500。**穩定度**：中(k 選擇敏感)。**誤導風險**：高。
- **白話**：α_tail 小(~2-3) → 極端事件相對常見(厚尾)。**禁止**：不得當崩盤預告。

### 3.10 Visibility Graph 指標 — D0 / B
- **數學定義**：自然可視圖：節點=每根 bar，i–j 連邊若中間所有點低於兩端連線。輸出度分布冪律指數 γ、平均度、assortativity。序列分形 ⇔ 度分布冪律。
- **最小資料**：N ≥ 200。**成本**：中(O(N log N) 有快演算法)。**白話**：γ 小/重尾 → 序列較分形、有樞紐結構。**禁止**：不得當訊號。

### 3.11 RQA（Recurrence Quantification）— D0 / B
- **數學定義**：延遲嵌入(維 m、延遲 τ 由 FNN/互資訊定)→ 遞迴矩陣 `R_ij=Θ(ε−‖x_i−x_j‖)`；輸出 RR、DET(對角線比例=決定論)、LAM(垂直線=層流/停滯)、L_max、ENTR(對角線長度熵)。
- **最小資料**：N ≥ 300。**成本**：高(O(N²))。**白話**：DET 高 → 結構較決定論/可重複；LAM 高 → 卡在某狀態。**禁止**：DET 高 ≠ 可預測獲利。

### 3.12 Avalanche 冪律指數 〔自組織臨界〕— D0/D1 / B
- **數學定義**：定義雪崩=連續同號報酬的累積幅度，或超過門檻的事件大小 s；用 MLE(Clauset–Shalizi–Newman)擬合 `P(s)~s^{-τ}` + KS 檢定 + x_min 選擇；報告 τ 與「是否通過冪律檢定」。可與神經雪崩 τ≈1.5 對照(實驗趣味)。
- **最小資料**：事件數 ≥ 100(常需 N ≥ 1000 或跨股票池)。**穩定度**：低-中。**誤導風險**：高。
- **白話**：接近冪律且 τ 在臨界帶 → 系統「接近臨界/自組織」。**禁止**：嚴禁當「快崩了」。需強免責。

### 3.13 Largest Lyapunov λ — D0 / B→D（Research）
- Rosenstein 法估最近鄰平均發散斜率；可預測地平線≈1/λ。金融資料雜訊大、**穩定度低**，列研究。**禁止**：不得當「可預測天數」對外宣稱。

### 3.14 TDA 持續同調 — D0/D1 / B（Research）
- 滑動視窗延遲嵌入點雲 → 持續圖(H0/H1) → 持續景觀/持續熵；H1 環=反覆體制。**成本高**、解釋成本高，列研究/實驗室。

### 3.15 跨股 — Cross-sectional Dispersion / 平均相關 — D1 / A→B
- **數學定義**：同一日 N 檔報酬的橫截面標準差(dispersion)，或平均成對相關 ρ̄(滾動視窗)。CSAD 亦可。
- **最小資料**：≥ 30 檔同步序列。**穩定度**：高。**白話**：ρ̄ 高 → 大家越來越像一顆腦(系統性風險上升的**描述**)。**禁止**：高相關 ≠ 會跌。

### 3.16 Kuramoto 同步序參數 r — D1 / B
- **數學定義**：每檔以 Hilbert 變換取相位 θ_j(t)(先帶通濾波 returns)；`r(t)=| (1/N)Σ_j e^{iθ_j(t)} |` ∈[0,1]。
- **最小資料**：≥ 30 檔、N ≥ 250。**成本**：中。**穩定度**：中。**白話**：r→1 全市場相位同步(羊群/極端一致)；r 低 各走各的。**禁止**：r 高 ≠ 泡沫破裂時點。

### 3.17 Transfer Entropy 網路 — D1 / B（Research）
- 估 `TE_{X→Y}` 方向性資訊流，建有向網路、算中心性。**成本高、穩定度低**，列研究。

### 3.18 敘事層 — Narrative Divergence / Entropy / Consensus / Information Flow — D2 / C
- **可觀測量**：新聞情緒分布、社群貼文情緒、搜尋趨勢、分析師預估離散度、籌碼/法人方向。
- **代理變數(OHLCV-only proxy，明確標為 proxy)**：
  - 敘事分歧 proxy = 日內(高-低)/隔夜跳空的離散度 + 量能波動(vol-of-vol)。
  - 共識 proxy = 連續同向 + 量增的程度。
- **哪些是假設**：把上述 proxy 當「信念分歧」是**假設**；真正測量需 D2 資料。
- **�random險**：高。列 Research/實驗室，重免責。

### 3.19 Tier D 探索（量子定價/波函數/Hopfield 盆地/集體意識/單一智慧體）— D2 / D
- **現況**：數學上可寫，但**缺乏可靠可觀測量**、難證偽、易被誤解。
- **缺什麼**：個體信念狀態、跨主體耦合矩陣、可重複的「測量」協定。
- **何時可升級**：若取得大規模逐筆+身份化下單流/社群情緒面板，部分(如信念場 proxy、Hopfield 體制吸引子估計)有機會升到 C。
- **定位**：Academic only，不進產品 UI。

## 4. 結構指紋卡（單一股票，D0/A 為主）

```
記憶性    ■■■■□□   ← DFA/Hurst H
複雜度    ■■■□□□   ← Permutation Entropy (+ SampEn)
湍流程度  ■■■■□□   ← Multifractal Width Δα (+ RV 百分位)
噪音性    ■■□□□□   ← Spectral Slope β（離白噪音多遠）
波動聚集  ■■■■■□   ← Volatility Clustering Index
```

## 5. 市場心智雷達（市場層級，Phase 2 / D1）

儀表板維度：
- **市場同步度**：Kuramoto r。
- **羊群程度**：平均成對相關 ρ̄ / dispersion。
- **市場熵**：橫截面報酬分布熵(或全體 PE 中位數)。
- **臨界性 / 雪崩風險**：雪崩冪律 τ + 是否通過冪律檢定 + 網路同步。
- **分形性**：全體 Hurst 中位數 / 譜斜率 / 可視圖。
使用者能看到：「現在市場像不像一顆同步的腦」「離臨界多近」「敘事多分裂」——全為**狀態描述**，每格附歷史百分位與免責。**不**輸出「該進該出」。

## 6. MVP 建議（第一版，5–7 個）

挑「成本最低 / 穩定最高 / 最好解釋 / 最不像訊號」：
1. **DFA / Hurst H**（記憶性）
2. **Permutation Entropy**（複雜度，最穩最便宜）
3. **Sample Entropy**（複雜度佐證）
4. **Spectral Slope β**（噪音顏色）
5. **Volatility Clustering Index**（波動聚集）
6. **Realized Vol 百分位**（湍流/快）
7. （加分）**Multifractal Width Δα**——僅在 N≥500 顯示，否則灰掉。

對映指紋卡：記憶性=1、複雜度=2+3、噪音性=4、波動聚集=5、湍流=6(+7)、臨界傾向(單股 proxy)=Tail/vol-of-vol(Phase 2 再補)。
**MVP 不含**：同步性(需 D1)、雪崩/臨界(穩定度低)、Lyapunov/TDA/敘事/量子。

## 7. 成熟度分級詳述（用「可觀測/可驗證/可重現/可推翻」分類）

每項格式：成熟度(1–5) ｜ 資料需求 ｜ 可驗證性 ｜ 可解釋性 ｜ 工程難度 ｜ 誤導風險 ｜ 建議定位。

### Tier A（可直接實作，工程級）
**為何可信**：僅需 OHLCV、文獻成熟、估計子有標準實作、可在合成資料(已知 H/熵)上回測驗證、可用人話講。
- **DFA/Hurst**：成熟 5｜D0｜驗證高｜解釋高｜工程低｜誤導中｜MVP。陷阱：短序列、趨勢污染。
- **Permutation Entropy**：5｜D0｜高｜高｜低｜低｜MVP。陷阱：d!≫N、平盤 tie。
- **Sample Entropy**：4｜D0｜高｜中｜中｜低｜MVP。陷阱：r 敏感。
- **Spectral Slope β**：4｜D0｜中｜中｜低｜中｜MVP。陷阱：頻帶/洩漏。
- **Volatility Clustering**：5｜D0｜高｜高｜低｜低｜MVP。
- **Realized Vol 百分位 / 短期自相關**：5｜D0｜高｜高｜低｜中｜MVP。
- **Multifractal Width**：3.5｜D0｜中｜中｜中｜中｜MVP-stretch(N≥500)。

### Tier B（可研究，實驗級）
**最小資料/驗證/失真**：多在 D0 但需長序列或 D1；以合成系統(Lorenz、SOC 沙堆、Kuramoto 模型)驗證估計子；失真來自非平穩、參數選擇、樣本數。
- **Tail Index(Hill)** 3｜D0｜中｜中｜中｜高｜Phase2。
- **Visibility Graph** 3.5｜D0｜中｜中｜中｜中｜Phase2。
- **RQA** 3.5｜D0｜中｜中｜高｜中｜Phase2。
- **Avalanche 冪律** 3｜D0/D1｜中(KS檢定)｜中｜中｜高｜Phase2。
- **Cross-sectional Dispersion/平均相關** 4｜D1｜高｜高｜中｜中｜Phase2。
- **Kuramoto r** 3.5｜D1｜中｜中｜中｜中｜Phase2。
- **Lyapunov / Transfer Entropy / TDA** 2.5｜D0/D1｜低-中｜低-中｜高｜中-高｜Research。

### Tier C（需代理變數，敘事層）
- **Narrative Divergence/Entropy/Consensus/Information Flow** 2｜D2｜低｜中｜高｜高｜Research。
- **可觀測量**：新聞/社群情緒、搜尋、分析師離散度、法人籌碼。**代理**：見 §3.18。**哪些是假設**：proxy↔信念的對應；**哪些是真量測**：情緒分數本身、籌碼數字本身。

### Tier D（理論探索，Academic only）
- **量子定價/波函數/Hopfield 盆地/集體意識/單一智慧體** 1｜D2｜低｜低｜高｜極高｜不進 UI。
- **缺什麼**：個體信念可觀測量、耦合矩陣、可重複測量協定。**何時可升級**：見 §3.19。

## 8. Codex 實作拆解

### Phase 1（MVP，D0/TierA）
- **Data Layer**：重用 `SQLiteStore` 取 OHLCV；新增「取最近 N 根 returns」工具。無新資料源。
- **Feature/Metrics Engine**：新檔 `app/analyze/structure_metrics.py`：`hurst_dfa()`, `permutation_entropy()`, `sample_entropy()`, `spectral_slope()`, `vol_clustering()`, `realized_vol_pct()`, `multifractal_width()`(可選)。每個回 `{value, series?, n, confidence, available, reason}`。加快取(以 stock_id+last_date+window 為 key)。
- **API**：`GET /api/stocks/{id}/structure`(回六維 + 原始值 + 充足度)。
- **Visualization**：個股頁「結構指紋卡」元件(六長條 + hover 白話/免責 + 充足度徽章)。
- **Report**：把指紋卡併入既有 HTML/Excel 報告。
- **驗收**：合成資料測試(白噪音→H≈0.5、PE≈1；fBm 已知 H；正弦+噪音→β)；金融資料冒煙測試；資料不足正確降級。

### Phase 2（市場層，D1/TierB）
- **Data Layer**：跨股票橫截面載入器(對齊交易日、≥30 檔、處理缺漏)。
- **Metrics**：dispersion/平均相關、Kuramoto r、雪崩冪律(MLE+KS)、Visibility/RQA(可選)。
- **Visualization**：「市場心智雷達」儀表板 + 歷史百分位。
- **驗收**：以模擬(Kuramoto 模型已知耦合、沙堆模型已知 τ)驗估計子。

### Phase 3（敘事層 + 研究，D2/TierC、Research）

- **Report Generator**：研究報告匯出(含方法、參數、信心、限制)。

## 9. 免責設計（產品內建）

1. **全域常駐標語**：「結構描述工具 · 描述現在 · 不預測未來 · 非投資建議」。
2. **每個指標**：附「白話解讀」與「**不得解讀為**」兩欄(見 §3)。
3. **充足度/信心徽章**：N 不足、R² 低、樣本少 → 顯示「資料少，僅供參考」並降低視覺權重(灰/虛)。
4. **分層閘門**：Tier C/D 收進「研究實驗室」，預設關閉，開啟前一次性說明。

6. **方法透明**：每個值可展開「怎麼算的 + 參數 + 限制」。

## 10. 最終架構建議（PM × 量化 × 架構師，務實版）

**只做一個真正有價值的版本 → 留這些：**
- 一個引擎 `structure_metrics.py`，**6 個 D0/TierA 描述子**(Hurst、PermEn、SampEn、β、Vol-Clustering、RV 百分位) + 可選 Multifractal Width。
- 一張**結構指紋卡**(六維、白話、充足度、免責)，掛在現有個股頁與報告。
- 完整免責與資料充足度閘門。

**理由**：只靠 OHLCV(資料你已有)、估計子可在合成資料上**驗證**、可用人話解釋、最不易被當訊號、維護面小、未來可平滑擴展到 Phase 2/3。

**砍掉 / 延後：**
- 砍(暫不進產品)：量子定價、波函數、Hopfield 盆地、集體意識、單一智慧體(Tier D)。
- 延後到 Phase 2：同步性(Kuramoto)、雪崩/臨界、跨股相關、市場雷達(都需 D1 橫截面載入器)。
- 延後到 Phase 3/研究：Lyapunov、TDA、Transfer Entropy、敘事層(需 D2 或重 proxy，穩定度低)。

**一句總結**：把「市場即心智」做成**一台只描述、不承諾的顯微鏡**——先穩穩交付 6 個看得懂、算得準、解釋得清的 OHLCV 結構描述子，其餘按 D0→D1→D2、A→B→C→D 逐層長出來。能做、能解釋、能維護、能長期演化。
