# 股票翻譯機 — 產品藍圖

> 給看不懂看盤軟體的散戶的「股票翻譯機」:把市場資料翻譯成人話,讓一般人第一次有能力自己驗證一支股票。
> 台股 · 本地優先(離線可看)· 不下單、不串券商、不預測股價。

---

## 一頁藍圖

| 面向 | 結論 |
|------|------|
| 目標使用者 | 台灣一般散戶——有買股票或想買,但看不懂專業看盤軟體 |
| 核心痛點 | 聽到明牌無法驗證;打開看盤軟體看不懂;不知道股票貴還便宜 |
| 核心功能 | **AI 白話健檢報告**:輸入股票代號,30 秒得到 4 維度人話摘要(趨勢、價格位階、獲利能力、波動風險),每個名詞點一下就有解釋 |
| 差異化 | 市面軟體比「資料多、圖表專業」;本產品比「看得懂」。資料可以少,每一筆都附白話解讀 |
| 商業紅線 | 只做「資料解讀」,絕不輸出買賣建議(台灣投顧法規,見《06》R1) |
| 平台 | 單機桌面應用(網頁技術構建);資料同步到本地資料庫,**離線照常可看** |
| MVP 時程 | 4 週,一人 + AI Coding Agent,W4 給同事試用收回饋 |
| 架構原則 | 模組化單體:四層(資料/分析/AI解說/UI)、目錄即模組、介面即契約,任一模組可獨立重寫 |

## 核心使用場景

朋友說「2330 可以買」→ 打開 App 搜尋 → 30 秒看完白話健檢報告 → 不懂的詞點一下 → 加自選股持續觀察。**從「聽明牌只能照單全收」變成「自己看得懂再決定」。**

## MVP 範圍(W1–W4)

搜尋個股 → 盤中報價顯示 → 走勢圖 + 基本資料 → AI 白話健檢報告(⭐)→ 名詞即點即解 → 自選股 → 每日資料同步與離線瀏覽。
明確不包含:自動下單、程式交易、高頻看盤、AI 報明牌、AI 預測股價。完整分級見《02》。

## 給 AI Coding Agent 的入口

開發前必讀順序:《04-技術架構》(模組邊界與契約)→《02-功能優先級》(只做 M1–M6)→《05》W1 任務。
鐵則:UI 不直連外部 API;AI 不算數字、不給建議;每模組附契約測試。

## 目前工程入口(W1 資料層)

已先用 Python 標準庫建立本地資料引擎雛形:

```powershell
python -m unittest discover -v
.\run_app.bat
python -m app.cli --db data/dev.sqlite3 sync-stock 2330 --lookback-days 365 --end-date 2026-06-12
python -m app.cli --db data/dev.sqlite3 show-stock 2330 --limit 5
python -m app.cli --db data/dev.sqlite3 search 台積
```

目前模組:

- `app/sync`:證交所資料轉接器與同步服務。
- `app/store`:SQLite 本地資料庫。
- `app/analyze`:純計算指標,目前包含價格摘要與健檢報告所需的趨勢/位階/波動。
- `app/explain`:白話解說層,目前先用規則式 fallback,不輸出買賣建議。
- `app/glossary`:靜態名詞字典,供 UI 點擊解釋專業詞。
- `app/portfolio`:持倉翻譯機純計算層,由交易紀錄推導持倉、成本、已/未實現損益。
- `app/quote`:盤中報價來源介面,目前先接 TWSE MIS,可替換成券商或資料商 API。
- `app/cli.py`:W1 驗證用命令列入口。
- `app/web` + `app/ui/static`:本機 Web UI 與 JSON API。
- `run_app.bat`:一鍵啟動 UI,預設開啟 `http://127.0.0.1:8765`。

目前 UI 已包含:

- 盤中報價與即時計算:顯示目前成交價;若成交價欄位暫無值,改用買一/賣一中間價並明確標示為參考。
- 明牌驗證工作台:先顯示趨勢、價格位階、資料缺口三句重點。
- 每月營收:同步 TWSE 月營收,顯示當月營收、月增率、年增率、累計年增與白話摘要。
- 基本面獲利能力:同步最新季綜合損益表與資產負債表,顯示 EPS、淨利率、單季 ROE、單季 ROA。
- 白話健檢報告:趨勢、價格位階、獲利能力、波動風險。
- 名詞即點即解:專業詞可點開白話說明。
- 自選股:加入/移除後會保存在本地 SQLite。
- 股利與估價試算:同步官方股利分派、本益比、殖利率、股價淨值比,並用殖利率法試算便宜價/合理價/昂貴價。
- 持倉翻譯機 MVP:手動新增/編輯/刪除交易紀錄,用移動平均成本法推導目前持倉、平均成本、估算市值、帳面損益與白話解讀;第一版明確標示不含股利、除權息、XIRR、稅務與多帳戶。

實作原則維持:外部資料只進 `sync`,寫入本地 SQLite 後才供查詢;CLI 的 `show-stock` 與 `search` 都只讀本地資料庫。
